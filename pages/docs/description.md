# 상품 목록 페이지 (Products List Page)
1) 페이지 개요
-목적: 상품을 빠르게 검색/필터/정렬하고, 일괄 작업(엑셀 내보내기, 일괄수정, 외부 송신, 삭제)을 수행한다.
-대상 데이터: 상품 카드(썸네일/이름/코드/상태/브랜드/카테고리/재고/가격/등록일), 옵션(행 확장 시).
-페이지네이션: 페이지당 20개 표시.
-보기 밀도: 간략히 보기 / 자세히 보기 전환(썸네일 크기, 행 간격, 폰트 크기 변경).

2) 상단 띠 영역(빠른 검색)
-구성: “통합검색” 라벨 + 검색 입력 + 검색 버튼.
-입력: 상품명, 상품코드 검색. 포커스 시 제안 목록(최근 검색어) 표시.
-실행: Enter 또는 “검색” 버튼.
-단축키: ⌘/Ctrl+K, / 로 검색 입력 포커스.
-지우기: X 버튼으로 즉시 초기화.
-오류/빈 값: 비어 있으면 검색 미실행(안내 없음).

3) 정렬/필터 토글
-정렬: 최신순 / 오래된순 / 가격 낮은순 / 가격 높은순. 드롭다운 변경 시 즉시 적용.
-필터 펼치기/접기: 버튼으로 패널 열고 닫기.

4) 적용된 필터 요약(칩)
-표시: 현재 적용된 필터를 칩으로 나열.
-해제: 각 칩의 X 클릭 시 해당 필터만 해제.
-전체 해제: “전체 해제” 클릭 시 모든 필터 초기화.
-포커스 이동: 칩 클릭 시 해당 필터 컨트롤로 포커스.

5) 필터 패널(펼침 시)
-공통: 미선택 허용(“전체/미지정”), 변경 즉시 목록 반영.
-상품 등록일자: 시작일~종료일(YYYY-MM-DD). 잘못된 범위는 경고 문구 표시.
-카테고리(계층형): 노드 선택 시 적용. 미선택=“전체”.
-브랜드: 등록된 브랜드 드롭다운. 미선택=“전체”.
-공급처: 다중 선택 칩. 0개 선택=“전체”.
-디자이너/등록자: 드롭다운(조직 사용자). 미선택 허용.
-시즌: 칩(“전체/SS/FW/AW” 중 1개).
-연도: 칩(“전체/올해/이전 연도들” 중 1개).
-상품 그룹: 다중 선택 드롭다운(체크박스). ‘전체’ 선택 제공, 패널 닫기 버튼 포함.
-배송비정책: 라디오(모든 상품 / 정책 있는 상품만).

6) 본문 검색 섹션(상세 검색)
-구성: “검색” 제목 + 통합검색 입력 + 검색 버튼(상단 띠와 동일 동작).
-제안: 최근 검색어 리스트(선택 시 즉시 적용).
-키보드: ↑/↓로 제안 이동, Enter 선택, Esc로 닫기.

7) 액션 툴바
-엑셀 다운로드: 현재 목록을 파일로 저장(파일명: products-list.xlsx).
-도움말: 기능 요약/사용 가이드 모달 또는 패널 표시.
-상품 일괄수정: 가격/상태 등 상품 단위 일괄 수정 화면으로 이동.
-파일 업로드 일괄수정: CSV 등 파일 기반 일괄수정 화면으로 이동(샘플/검증 안내 권장).
-옵션 일괄수정: 옵션 단위 가격/재고 일괄 변경 화면으로 이동.
-판매처 선택 + 선택 외부 송신: 드롭다운에서 판매처 선택 후 “선택 외부 송신” 클릭 → 확인 모달에서 전송.
-선택삭제: 선택 항목 휴지통 이동(확인 팝업).

8) 결과 테이블 — 헤더
-전체 선택 체크박스: 현재 페이지 항목 전체 선택/해제(0개면 비활성화).
-컬럼 구성: 선택 / 상품정보 / 카테고리·브랜드 / 재고 / 가격 / 등록일 / 액션.

9) 결과 테이블 — 행(상품)
-선택 체크박스: 개별 선택/해제.
-썸네일: 이미지 없으면 기본 아이콘. 간략 48px / 자세히 120px.
-상품정보:
-좌측: 옵션 펼치기 토글(▶/▸), #ID, 상품명(없으면 “이름 미입력”), 상품코드(복사 버튼, “복사됨” 힌트).
-우측: 판매상태(판매중/판매중지), 활성상태(활성/비활성), 가격(콤마 표기).
-행 클릭 시 상세로 이동(단, 입력/버튼/링크 클릭은 이동 제외).
-카테고리/브랜드: 텍스트 표시(없으면 “카테고리 미지정” / “브랜드 미등록”).
-재고: 총 재고 칩 — 색상 규칙 0=빨강, 1~5=주황, 6+=중립. 값 없음/확인 불가 → ‘재고 확인 필요’.
-가격: 판매가(콤마 표기). 값 없음은 “—”.
-등록일: 상대 표시(방금 전/분/시간/일 전), 마우스오버 시 상세 시간 툴팁.
-액션 버튼: 편집 / 외부 송신 / 삭제
-편집: 상세 이동.
-외부 송신: 판매처 선택 모달 오픈 후 전송.
-삭제: 확인 후 휴지통 이동.

10) 행 확장(옵션 목록)
-펼침: 상품행의 토글 클릭 시 옵션 테이블 표시(부모행 바로 아래).
-옵션 테이블 컬럼: 옵션명/코드 / 판매상태·재고 / 위치 바코드 / 바코드 / 규격·중량 / 최종수정일 / 작업.
-인터랙션:
-옵션 행 클릭 시 옵션 상세 이동(입력/버튼 클릭 시 이동 제외).
-판매상태: 버튼 토글(판매중 ↔ 상태 미확인).
-재고: 숫자 인라인 편집(0 이상).
-바코드: 일부 표시, 자세히 보기/드로어로 전체 확인.
-빈 값: “—” 또는 “미지정” 표기.

11) 외부 송신 모달
-열림: “선택 외부 송신” 또는 행의 “외부 송신” 클릭 시 오픈.
-내용: 대상 판매처 드롭다운 + 취소/전송 버튼.
-포커스: 모달 내부 포커스 트랩, Esc로 닫기. 닫힘 시 이전 포커스로 복귀.
-검증/오류:
-선택 상품 없음: “전송할 상품을 선택하세요.”
-판매처 미선택: “전송할 판매처을 선택하세요.”
-전송 중 상태: “외부로 전송하는 중…”
-성공/실패: 토스트로 결과 안내.

12) 페이지 상태
-로딩: “불러오는 중… 잠시만 기다려주세요.”
-빈 상태(검색 결과 0개):
-제목: “조건에 맞는 상품이 없습니다.”
-설명: “새 상품을 등록하거나 가져오기를 통해 데이터를 추가하세요.”
-행동: “신규 상품 등록”, “상품 가져오기” 버튼.
-토스트 메시지(예시):
-선택 삭제 전: “삭제할 상품을 선택하세요.”
-단건 삭제 후: “상품이 휴지통으로 이동했습니다.”
-외부 송신 오류: “전송 중 오류: {사유}”

13) 선택/일괄 동작 규칙
-전체 선택: 현재 페이지 항목만 선택.
-선택 유지: 필터/정렬 변경 시에는 결과에 따라 선택 상태가 변경될 수 있음(권장: 변경 후 사라진 항목은 선택 해제).
-선택삭제: 휴지통(Local)로 이동(소프트 삭제), 목록에서 즉시 제외.

14) 키보드/접근성
-단축키: ⌘/Ctrl+K, / 검색 포커스. 모달 Esc 닫기. 제안 목록 ↑/↓ 이동, Enter 선택.
-포커스: 모달 오픈 시 내부 포커스 유지, 닫을 때 이전 포커스로 복귀.
-스크린리더: 버튼/입력/라디오/체크박스는 라벨 및 상태(선택/미선택) 읽힘. 오류 메시지는 해당 필드와 연동.

15) 표시/포맷 정책
-금액·수량: 천 단위 콤마(예: 12,000).
-길이·무게·부피: 단위 라벨 고정 노출(예: cm, kg, L).
-날짜: 목록은 상대 시간, 툴팁으로 상세 시간.
-빈 값: “—” 또는 “미지정” 통일 표기.
-코드 복사: 복사 성공 시 1.5초간 “복사됨” 힌트 표시.

16) 공통 입력 규칙(검색/텍스트 필드에 적용)
-허용 문자: 한글, 영대소문자, 숫자, 공백, 사용 가능한 특수문자 32자
! " # $ % & ' ( ) * + , - . / : ; < = > ? @ [ ₩ ] ^ _ \ { | } ~ (역따옴표 ` 포함)
-실시간 유효성 검사:
-허용되지 않는 문자 입력 시 필드 하단 에러: ‘{입력한 값}’은 유효하지 않은 문자입니다.
-필수값 미입력:
-저장 시 상단 배너 + 해당 필드 하단 에러 동시 표시(검색은 필수 아님).

17) 에러/안내 메시지(샘플)
-검색 입력: 허용되지 않는 문자 사용 시 → ‘{입력한 값}’은 유효하지 않은 문자입니다.
-삭제: “정말 이 상품을 휴지통으로 이동하시겠습니까?”(확인 팝업).
-외부 송신:
-“전송할 상품을 선택하세요.”
-“전송할 판매처을 선택하세요.”
-“외부로 전송하는 중…”
-“전송 중 오류: {사유}”
-“{N}개 상품을 {판매처명}으로 외부로 전송했습니다. (목업)”


# 상품 등록/수정 페이지 (Product Add Page)
상품명(필수) : 텍스트 필드, 최대 255자, 한글/영문/숫자/특수문자 32종 입력 가능
-텍스트 입력 길이: 2 ~ 255자
-입력 가능 문자: 한글, 영대소문자, 숫자, 공백, 특수문자 32자
( ! “ # $ % & ’ ( ) * + , - . / : ; < = > ? @ [ ₩ ] ^ _ ` { | } ~ )
-실시간 유효성 검사
-잘못된 문자 입력 시: ‘{입력한 값}은 유효하지 않은 문자입니다.’
-글자 수 미만/초과 시: ‘2~255자 사이로 입력하세요.’

사입상품명 : 텍스트 필드, 최대 255자
-기본값: {상품명}
-동작: 상품명 입력 후 포커스 아웃 시 자동으로 동일 값 기재
-사용처 예: 발주서/사입 내역 등 내부 관리명

상품코드(필수) : 텍스트 필드, 최대 50자(권장), 영문/숫자/특수문자 일부 허용
-텍스트 입력 길이: 1 ~ 50자 (운영 편의를 위한 권장 범위)
-입력 가능 문자: 영대소문자, 숫자, -, _, .
-정책: 중복 불가(이미 존재하는 코드 사용 금지)
-실시간 유효성 검사
-미입력 시: ‘상품코드는 필수 항목입니다.’
-허용 외 문자 포함 시: ‘허용되지 않는 문자가 포함되어 있습니다.’
-중복일 경우: ‘이미 사용 중인 코드입니다. 다른 코드를 입력하세요.’

대표 이미지(필수) : URL 입력 + 목록 관리
-입력 방식: 이미지 URL 붙여넣기 후 Enter 키 → 목록 상단에 추가
-기본값/자동: 첫 번째 추가 이미지는 자동으로 ‘대표’ 지정
-관리: 각 이미지에 대해 [대표 지정] / [삭제] 가능
-표시: 썸네일 미리보기 제공, 최소 1개 이상 필요
-허용 포맷/권장 크기: JPG/PNG, 800px 이상 권장
-오류 안내: 잘못된 URL 형식일 경우 ‘유효한 이미지 주소를 입력하세요.’

카테고리(필수) : 계층 선택(최대 3단계)
-선택 방식: 트리형 선택(최대 3뎁스), 선택 경로 자동 저장(예: 의류 > 남성 > 셔츠)
-표시 값: 선택 경로를 ‘ > ’로 이어서 표시
-실시간 유효성 검사
-미선택 시: ‘카테고리를 선택하세요.’
-추가 입력(선택): ‘상품 그룹’ 필드에서 ‘> , / \’ 구분자로 직접 경로 입력 가능
-예: “의류 > 남성 > 셔츠” 입력 시 동일 경로로 저장

브랜드(필수) : 드롭다운
-선택 방식: 목록에서 1개 선택
-실시간 유효성 검사
-미선택 시: ‘브랜드를 선택하세요.’

공급사 : 드롭다운(선택)
-선택 방식: 목록에서 1개 선택
-안내: 미등록 공급사는 사전 등록 후 선택 가능

상품 일련번호 : 텍스트 필드(숫자 권장)
-용도: 내부 품목 식별용
-검증: 숫자 외 문자가 포함되면 ‘숫자만 입력 가능합니다.’ 안내

대표 판매가(필수) : 숫자 입력(원)
-입력 형식: 0 이상 정수(원 단위)
-실시간 유효성 검사
-미입력/0 미만 시: ‘대표 판매가를 올바르게 입력하세요.’
-안내: 가격은 원 단위로 입력, 소수점 미사용

대표 공급가(필수) : 숫자 입력(원)
-입력 형식: 0 이상 정수(원 단위)
-실시간 유효성 검사
-미입력/0 미만 시: ‘대표 공급가를 올바르게 입력하세요.’

마진 금액 : 읽기 전용
-계산식: 마진 금액 = 대표 판매가 − 대표 공급가
-동작: 판매가/공급가 변경 시 즉시 재계산

판매처별 상품코드 : 멀티라인 텍스트(줄바꿈 구분)

-예)
Cafe24: ABC-001
SmartStore: SNS-001

-중복 라인 처리: 동일 채널 코드가 여러 줄일 경우 마지막 입력 라인을 우선 저장(저장 시 경고 문구 권장)

편직 정보 : 텍스트 필드(선택)
-예: ‘니트 12G’, ‘우븐’ 등 제조/편직 특성 기재

영문 카테고리명 : 텍스트 필드(선택)
-예: ‘Outer > Coat’

세탁방법 : 텍스트 필드(선택)
-예: ‘드라이클리닝 권장’, ‘찬물 손세탁’ 등

영문 상품 카테고리명 : 텍스트 필드(선택)

면세 코드 : 텍스트 필드(선택)
-안내: 면세 및 통관 관련 내부 식별용

브랜드 수수료율(%) : 숫자 입력(0~100)

# 외부 판매처 상품 정보 불러오기 (Import Products from External Marketplaces)

[페이지 정보]
-페이지명: 외부 판매처 상품 정보 불러오기
-목적: 여러 판매처(카페24/메이크샵/스마트스토어/고도몰 등)에서 상품 정보를 가져와 통합 관리
-대상: 상품/운영 담당자, 통합몰 운영자
-기본 흐름: (1) 필터 설정 → (2) 활성 필터 확인/수정 → (3) 탭 이동(개요/연결된 판매처) → (4) 판매처별 가져오기/동기화 실행 → (5) 결과 확인(성공/실패/기록)


[화면 구조]
-헤더: 제목(외부 판매처 상품 가져오기) + 한 줄 설명
-필터 패널: 판매처/기간/상태/검색어 + 옵션(자동 분류 등록, 옵션명 적용) + 빠른 기간 버튼 + 활성 필터 칩
-탭: 개요 / 연결된 판매처 / 가져오기 기록 / 오류 상품
-하단 일괄 작업: 전체 판매처 동기화, 로그 보기, 가져오기 기록, 오류 상품 관리


[필터 패널 — 항목별 정의]
	1.	판매처 선택

-유형: 드롭다운 + 상단 빠른 선택 버튼
-기본값: 전체(“all”)
-동작: 특정 판매처 선택 시 해당 판매처 기준으로 필터 적용
-오류/안내: 미연결 판매처에서 “가져오기” 실행 시 → ‘먼저 판매처을 연결해주세요.’

	2.	기간 선택(From/To)

-형식: YYYY-MM-DD
-빠른 기간 버튼: 오늘 / 7일 / 10일 / 15일 / 1개월 / 3개월 / 6개월(클릭 즉시 채움)
-오류/안내: 시작일 > 종료일일 경우 → ‘기간을 올바르게 선택하세요.’

	3.	등록일자 영역(전체/적용/초기화)

-전체: 기간 값을 모두 비움(전체 포함)
-적용: 현재 필터로 검색(시연용 안내 가능)
-초기화: 판매처/기간/상태/검색어/옵션 등 모든 필터 기본값으로 재설정

	4.	원가/판매가 수정여부

-유형: 단일 선택(수정함 / 수정안함)
-설명: 선택한 범위 내 가격 수정 이력 기준으로 필터

	5.	진열여부

-유형: 단일 선택(진열함 / 진열안함)

	6.	판매여부

-유형: 단일 선택(판매함 / 판매안함)

	7.	상품명

-유형: 텍스트 입력(부분 일치 검색)
-권장 입력 길이: 2~255자
-오류/안내: 2자 미만의 짧은 검색어는 결과가 부정확할 수 있음

	8.	상품분류 자동 등록

-유형: 체크박스(On/Off)
-설명: 체크 시 가져오기 시점에 분류가 없으면 자동 생성/등록(사내 분류 정책과 충돌 시 운영 정책 우선)

	9.	옵션명 적용

-유형: 체크박스(On/Off)
-저장 형식: “색상,사이즈=빨강,M” 형태로 옵션명 저장

[활성 필터 칩 — 공통 동작]
-칩 클릭: 해당 입력으로 즉시 포커스 이동(필터 패널 자동 펼침)
-칩 닫기(×): 해당 조건만 즉시 해제
-표시 수: 최대 6개, 초과 시 “+N” 요약칩 표시


[탭 — 개요(지표 정의)]
-총 연결 판매처: 현재 연결됨 상태의 판매처 수
-총 상품 수: 연결된 판매처의 상품 수 합계(천 단위 구분)
-최근 가져온 상품: 최근 가져오기 성공 건 합계

[탭 — 가져오기 기록(표 컬럼 예시)]
-컬럼: 일시 / 대상(판매처명) / 총건 / 성공 / 실패 / 소요시간
-정렬: 일시 내림차순(최신순)
-필터: 기간 / 판매처 / 결과(성공/실패)

[탭 — 연결된 판매처(목록 카드: 접힘 상태)]
-구성: 판매처명 + 상태 배지(연결됨/미연결/오류) / 설명 / 상품 수 / 버튼(가져오기, 동기화 또는 연결하기) / 펼치기 토글(+/−)
-상태 배지 규칙:
-연결됨(녹색): 정상 연동 가능
-미연결(회색): 연결 필요
-오류(빨간색): 연동 오류(점검/재설정 필요)

[탭 — 연결된 판매처(상세: 펼침 상태)]

A) 기능 및 설정
-실시간 동기화(체크박스): 체크 시 상품 정보 자동 동기화(사내 정책과 일치해야 함)
-기본값: 연결된 판매처는 체크 상태일 수 있음
-주기: 시스템 정책에 따름(예: 15분)

B) 가져온 상품 미리보기(표)
-목적: 최근 가져온 일부 상품을 빠르게 미리보기(최대 5행 권장)
-컬럼 정의:
-상품번호: 외부 판매처 상품 고유번호(원본 그대로 표시)
-상품명: 외부 판매처 상품명(긴 텍스트 말줄임 가능)
-가격: 원 단위, 천 단위 구분
-재고: 숫자(음수 불가)
-상태: 판매중 / 판매중단
-마지막 업데이트: YYYY-MM-DD HH:mm 형식

C) 가져오기 통계
-지표: 성공 건수 / 실패 건수 / 마지막 가져오기 시각
-정의: 성공=가져오기 처리 완료, 실패=오류로 미반영(사유는 기록/오류 탭 참고)

D) 상세 버튼
-전체 상품 가져오기: 해당 판매처 전체 대상
-신규/변경 상품만 가져오기: 증분 동기화
-기존 상품 동기화: 이미 가져온 상품을 최신값으로 동기화
-진행 표시: ‘상품 가져오는 중… {n}%’(진행 중에는 버튼 비활성화)

E) 지원 기능(칩)
-예시: 가격 동기화 / 카테고리 매핑 / 재고 관리 / 실시간 동기화 등


[하단 일괄 작업]
-전체 판매처 동기화: 연결된 모든 판매처를 한 번에 동기화
-동기화 로그 보기: 최근 동기화 결과/오류 내역
-가져오기 기록: 과거 작업 이력(일자/대상/건수/결과)
-오류 상품 관리: 실패 건 재처리/수정(사유별 묶음, 재시도/내보내기)


[안내/정책(고정 문구 — 카페24 자동등록)]
-상품명과 옵션명이 중복될 경우 등록되지 않습니다.
-상품코드가 중복될 경우 등록되지 않습니다.
-상품명은 존재하나 옵션이 없을 경우, 해당 상품에 옵션만 추가 등록됩니다.


[상태/오류 — 표준 문구]
-연결 필요: ‘먼저 판매처을 연결해주세요.’
-가져오기 완료: ‘{판매처명} 상품 가져오기가 완료되었습니다.’
-가져오기 실패: ‘가져오기 실패: {사유}’
-진행 중: ‘상품 가져오는 중… {n}%’
-통계 로딩: ‘통계를 불러오는 중…’


[사용성/접근성 규칙]
-활성 필터 칩 클릭 시 필터 패널 자동 펼침 + 해당 입력으로 포커스 이동
-숫자/날짜 입력 오류 즉시 안내(필드 하단)
-버튼 라벨은 ‘행위+대상’ 형태(예: 전체 상품 가져오기)
-긴 표는 가로 스크롤 허용, 헤더 고정 권장


[권한/전제 조건]
-연결 상태(토큰/계정) 유효 시에만 가져오기/동기화 가능
-권한 예시: 읽기(현황/통계/미리보기), 실행(가져오기/동기화), 설정(실시간 동기화/연결)


[라벨/용어 사전(화면 문구 그대로 사용)]
-탭: 개요 / 연결된 판매처 / 가져오기 기록 / 오류 상품
-버튼: 가져오기 / 동기화 / 연결하기 / 전체 / 필터 적용 / 초기화 / 전체 판매처 동기화 / 동기화 로그 보기 / 오류 상품 관리
-상태 배지: 연결됨 / 미연결 / 오류
-표 컬럼: 상품번호 / 상품명 / 가격 / 재고 / 상태 / 마지막 업데이트


[버튼 문구 원칙]
-버튼은 사용자가 무엇을 하게 되는지 즉시 이해할 수 있는 행위 중심 문장 사용
-진행/완료/실패 상태에 따라 라벨 또는 안내 토스트를 간결하게 노출


# 차수별 상품 등록 내역 (Batch Product Import History)
[페이지 정보]
-페이지명: 차수별 상품 등록 내역
-목적: 기간/입력처(플랫폼)별 상품 등록 작업을 차수 단위로 조회하고, 각 차수의 성공/실패 현황과 상세 리스트를 확인
-대상: 상품/운영 담당자, 통합몰 운영자
-기본 흐름: (1) 입력처·기간 선택 → (2) 검색/리셋 → (3) 목록에서 차수별 현황 확인 → (4) 리스트보기로 상세 상품 확인 → (5) 페이지네이션으로 이전/다음 차수 이동


[상단 컨트롤 바 — 필터/검색]
	1.	입력처(플랫폼)

-유형: 드롭다운
-값 예시: 카페24 / 메이크샵 / 고도몰 / 스마트스토어 / 위사몰 / CSV / 전체 플랫폼(기본)
-동작: 선택 시 해당 입력처의 차수만 표시

	2.	기간(From/To)

-유형: 날짜 입력(YYYY-MM-DD)
-동작: From 또는 To 비워두면 열린 구간으로 검색
-오류 문구: 시작일이 종료일보다 늦을 경우 → ‘기간을 올바르게 선택하세요.’

	3.	검색 / 리셋

-검색: 현재 조건 적용 후 1페이지로 이동
-리셋: 입력처/기간 초기화 + 1페이지로 이동


[결과 테이블 — 헤더]
-컬럼: 입력차수 / 입력처 / 건수 / 비고


[결과 테이블 — 행 정의]
	1.	입력차수

-표시: “{n} 차” + 하단에 등록 일시(브라우저 로컬 형식, 예: 2025.09.26. 14:30:12)
-정렬: 기본은 최신 차수부터(권장)

	2.	입력처

-표시: ‘카페24 상품등록’, ‘메이크샵 상품등록’ 등 입력처+행위 조합

	3.	건수

-총 건수: “총 {total}건”
-진행 바: 성공 비율(성공/총건)을 막대로 시각화
-실패 배지: 실패 건수 > 0일 때 붉은 배지로 ‘{failed}’ 표시
-텍스트 요약: “성공 {success} / 실패 {failed}”
-실패 배지(클릭 시): 오류 사유 요약 드로어 열림(선택 적용)

	4.	비고
